##Установка git
##Регистрация/авторизация
##Создание репозитория ##Команда для создания репозитория Git — git init. Эта команда инициализирует пустой репозиторий.
git clone	##Копирование репозитория
git add File1.txt ##Указание на существование файла
git status ##Просмотр папки git
git commit -m "commit файлов" ##Коммит файлов
git push ##Загрузка изменений в сеть
cat File1.txt ##Просмотр файла
git fetch ##это команда, которая используется для извлечения всех изменений из удалённого репозитория в локальный репозиторий.Она не вносит изменений в текущий рабочий каталог и сохраняет все изменения в отдельной ветке, называемой веткой удалённого отслеживания.Для слияния этих изменений в текущий рабочий каталог используется команда git merge или git rebase.
##origin main - версия ветки на удаленном репозитории
##origin/main - локальная, промежуточная ветка
git pull ##используется для извлечения изменений из удалённого репозитория и автоматического объединения их с текущей веткой. По сути, git pull — это комбинация команд git fetch и git merge
git log ##Просмотр всех commit
